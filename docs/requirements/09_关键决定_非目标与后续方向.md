# 09 关键决定、非目标与后续方向

## 1. 四句总边界

```text
用户负责真实探索
Capture 负责纯代码忠实记录
外部 AI 负责分析与推理
未来其他工具负责重放、遍历与自动化
```

## 2. v1 正式非目标

以下内容正式锁定，不得因为“顺手”进入 v1。

1. 不接入任何 LLM。
2. 不自动分析 Endpoint 重要性。
3. 不推断参数血缘和级联因果。
4. 不自动生成 Skill。
5. 不自动操作、遍历或穷举网页。
6. 不执行 Request Replay。
7. 不自动执行生成的 cURL。
8. 不自动验证 URL 是否稳定可复用。
9. 不做截图、录像、视觉分析。
10. 不保存完整 DOM。
11. 不建设 Selector / XPath Replay 系统。
12. 不建设完整 DevTools / Network 工作台。
13. 不做多 Session Diff 或 Comparison Bundle。
14. 不支持并发 Session。
15. 不支持并发 / 嵌套 Step。
16. 不支持 Session Pause。
17. 不允许事后调整 Step 时间边界。
18. 不做动态跨 Origin 探索。
19. 不建设 Raw 加密和密钥管理。
20. 不建设复杂 Session 标签、分类、知识库。
21. 不集成 Dify。
22. 不在本轮修改 tingyun-cli。

额外锁定：

23. 不兼容旧版 Capture 产物。
24. 不迁移旧 schema。
25. 不支持 Windows / Linux 正式验收。
26. 不打包 macOS 原生桌面 App。
27. 不支持局域网访问。
28. 不建设登录与权限系统。

## 3. 不做截图的影响

v1 的证据链是：

```text
用户意图
→ 关键页面操作
→ 表单状态
→ Tab / URL 变化
→ 网络 Request / Response
→ 用户备注
```

没有：

- 自动截图。
- 手工截图。
- 视频。
- 视觉分析。

后续若出现真实需求再评估。

## 4. 不做 Replay 的影响

v1 虽不执行 Replay，但必须保留未来可能需要的基础事实：

- Method。
- 完整 URL。
- Query。
- Headers。
- Request Body。
- 原始 Response。
- 时间上下文。

Review 提供参考 cURL 只是便利转换，不代表 Replay 能力。

## 5. 不做自动遍历的影响

表单“所有选项”只指：

> 本次真实探索实际暴露出来的所有选项。

未来如果需要自动穷举：

- 应作为新的自动探索工具。
- 必须与 Capture 事实记录职责分离。

## 6. 不做 AI 分析的影响

Capture 的 AI 责任只到：

- 材料准备好。
- 阅读入口准备好。
- 证据引用准备好。
- 分析任务说明准备好。

不负责：

- 选择模型。
- 调用模型。
- 验证模型结论。
- 管理 Prompt。

## 7. 未来可能方向

这些只是后续候选，不属于 v1 承诺。

### 7.1 外部 Session 分析 Skill

输入：AI-ready 包。

输出：

- 候选 Endpoint。
- 参数血缘。
- 级联链。
- 可复用探索路径。
- URL 候选。

### 7.2 Replay / 验证工具

可能基于 Capture Request Facts：

- 生成可执行请求。
- 更新认证。
- 替换参数。
- 验证返回。

必须独立于 Capture。

### 7.3 自动探索器

未来可：

- 遍历下拉选项。
- 执行受控实验。
- 验证级联。

但其产生的操作必须明确标记为自动化，而不能混入“用户真实操作”的 Capture 证据。

### 7.4 知识目录

外部 AI 可以把多个 Session 分析结果沉淀为：

- Endpoint 知识。
- 参数关系。
- 页面路径。
- 能力文档。
- Skill。

这些不是 Capture 的内部数据库职责。

## 8. Greenfield 原则

新版从空项目开始。

旧版：

- 删除。
- 不兼容。
- 不导入。
- 不迁移。

需求经验保留在本需求包中。

## 9. 范围变更规则

任何新能力进入 v1 前，应回答：

1. 是否直接服务 Endpoint、参数、级联、提交、探索路径或 URL？
2. 是否必须由 Capture 本身完成？
3. 是否可以由外部 AI、未来 Replay 工具或测试工具完成？
4. 是否会引入新的用户概念或复杂状态机？
5. 是否有真实使用证据证明需要？

若答案不充分，默认不加入。
