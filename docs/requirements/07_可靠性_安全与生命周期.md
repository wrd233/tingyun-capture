# 07 可靠性、安全与生命周期

## 1. 总原则

> 宁可诚实中断，也不能悄悄制造看似完整、实际残缺的 Session。

可靠性优先级：

1. 已发生事实尽快落盘。
2. 无法继续保证高保真时停止录制。
3. 明确记录中断或缺口。
4. 派生产物失败不能污染 Raw 状态。

## 2. 全局状态机

主状态：

```text
IDLE
  ↓
ACTIVE
  ↓
FINALIZING
  ↓
SEALED
```

异常路径：

```text
ACTIVE
  ↓ abnormal
INTERRUPTED
  ↓ user seals
SEALED
```

### 2.1 约束

- 全局只允许一个尚未完成处理的 Session。
- FINALIZING 完成前不能开始新 Session。
- 未处理 INTERRUPTED 存在时不能开始新 Session。

## 3. Step 生命周期

- 一个 Session 同时最多一个 Active Step。
- Step 开始必须有一句意图。
- Step 结束时间立即固定。
- 不等待网络静止。
- 不支持 Pause。
- 不支持嵌套。
- 不支持并行。
- 不支持事后改边界。

## 4. Crash Recovery

Capture 异常退出后重启：

```text
原 ACTIVE Session
→ 恢复为 INTERRUPTED
```

允许：

- 查看已有数据。
- 修改人工文字。
- 封存。
- 删除。

不允许：

```text
INTERRUPTED → ACTIVE
```

继续探索必须新建 Session。

## 5. Browser 关闭

Active Session 期间专用 Chromium 被关闭：

```text
Session = INTERRUPTED
reason = browser_closed
```

不自动重拉并续接原 Session。

## 6. Engine 主动停止

用户 Ctrl+C 或 stop：

如果存在 Active Session：

```text
Session = INTERRUPTED
reason = engine_stopped
```

正常结束必须先由用户在 Sidecar 明确 End Session。

停止整个 Capture 工具时：

- 关闭专用 Chromium。
- 保留持久化 Profile。

## 7. Sidecar 关闭

Sidecar 页面关闭或刷新：

- 不影响 Engine。
- 不影响 Browser。
- 不影响 Active Session。
- 不影响 Active Step。

## 8. 磁盘空间

### 8.1 定期检查

使用纯代码检查输出磁盘剩余空间。

### 8.2 低空间

显示明显警告：

```text
磁盘空间不足
当前 Session 仍在录制
请尽快结束 Session 或清理空间
```

继续录制。

### 8.3 危急空间

在真正写满前：

```text
停止接收新事实
→ 尽力保全已落盘数据
→ Session = INTERRUPTED
→ reason = low_disk_space
```

不做：

- 自动删除旧 Session。
- 静默停止 Body。
- 元数据继续录但假装完整。

具体阈值在设计阶段确定。

## 9. 关键持久化失败

### 9.1 适用场景

例如：

- events.jsonl 本应写入但失败。
- Body 本应保存但文件写失败。
- 关键元数据无法持久化。

### 9.2 行为

允许实现层进行极少量瞬时重试。

仍失败：

```text
停止接受新 Session 事实
→ Session = INTERRUPTED
→ reason = persistence_failure
```

### 9.3 与预期缺口区分

预期内：

```text
Body 超过上限
→ 明确 gap
→ Session 可继续
```

非预期：

```text
本应写入却失败
→ 无法保证高保真
→ Session 中断
```

## 10. Interrupted 的派生产物

异常中断时：

- 不自动生成 AI-ready。
- 不在异常现场继续做派生任务。

用户后续封存时再生成。

## 11. Session Finalization 失败与 AI-ready 失败

### 11.1 Raw 收尾成功

只要 Raw 正常收尾：

```text
Session = SEALED
```

### 11.2 AI-ready 失败

独立状态：

```text
AI-ready = FAILED
```

允许之后重新生成。

Session 不能因此变成 INTERRUPTED 或永远卡在 FINALIZING。

## 12. 本地安全边界

### 12.1 服务

仅绑定：

```text
127.0.0.1
```

### 12.2 Raw

Raw 可能包含：

- Authorization。
- Cookie。
- Token。
- 业务数据。
- 内网 URL。

因此：

- 默认私有。
- 明显标记 PRIVATE。
- 不提供一键分享。
- `.gitignore` 防止误提交。
- 尽量限制当前用户权限。

### 12.3 不做加密

v1 不建设：

- Raw 加密。
- 密钥管理。
- 解锁流程。
- 恢复密钥。

安全边界就是本机当前用户。

## 13. Review 安全

本地 Review 默认展示 Raw，因为其职责就是确认真实证据。

界面需要显式标识：

```text
RAW · PRIVATE · 本地私有证据
```

默认复制 cURL 仍脱敏。

## 14. AI-ready 安全

AI-ready：

- 高置信度凭据脱敏。
- 无法可靠脱敏的二进制不纳入。
- ZIP 永不包含 Raw。

## 15. 删除

允许用户删除：

- SEALED。
- INTERRUPTED。

删除整个 Session 目录和派生产物。

不做回收站。

## 16. Schema 兼容

每个 Session 自带 Raw schema 版本。

新版本：

- 尽量兼容读取。
- 不原地迁移封存 Raw。
- Derived 可以重新生成。

## 17. 关闭顺序

正常停止建议：

```text
用户 End Session
→ FINALIZING
→ SEALED
→ 停止 Capture
→ 关闭 Dedicated Chromium
```

异常直接 stop：

```text
ACTIVE
→ INTERRUPTED
→ 关闭 Browser
→ 停止 Engine
```
